Color rationale

Why we lead with a brighter palette instead of deep navy.

Light-first keeps us clear, open, and differentiated while still using Kalista Blue for emphasis and dark sections when we want drama.

Legibility & comfort

  • Dark text on light backgrounds reads cleanly without glow or halation.
  • Headlines, sub-copy, and nav items stay consistent against subtle gradients or patterns.

Signal & tone

  • Lighter surfaces feel open and transparent—aligned with “glass-box” AI.
  • Avoids the heavy “cyber/fintech neon” trope that can read aggressive.

Differentiation

  • Breaks from the sea of dark-blue AI landers; Kalista Blue becomes memorable, not generic.
  • We can still use dark bands strategically for emphasis without being locked into them.

Show the product

  • Light gradients let real UI captures, code, and diagrams sit cleanly on the page.
  • Abstract art is optional, not required to make text readable.

Versatility & ops

  • Print, slides, PDFs, and email templates stay legible and low-ink.
  • Developers still get dark UI where it belongs (app, playground, code), while marketing stays welcoming.

Typography

Inter for UI, JetBrains Mono for code.

Minimal weights for performance: 400 for body, 600-700 for headings, monospace at 14px for code.

UI copy

Inter Bold - H1 48px / 1.2

Inter Semibold - H2 36px / 1.25

Inter Regular - Body 16px / 1.5. Keep line length 60-75 characters for comfortable reading.

Use near-black text (#212121) on light backgrounds. Optional color accents for headings only.

Code and inline

// JetBrains Mono 14px
const client = new Kalista()
client.generate({ name: "Docs Theme" })

Inline code uses the same font with a subtle gray highlight. Avoid teal or orange for full blocks; reserve them for syntax accents.

UI components

Buttons, inputs, cards, and navigation on an 8px grid.

Spacing multiples of 8px

Buttons

Primary Secondary CTA
  • Radius 4px, 12/24px padding, Inter Semibold 14-16px.
  • Hover: deepen shade or add light shadow. Focus: 2px teal outline.

Inputs

  • Border #CCCCCC; focus border teal with subtle glow.
  • Use #888888 for placeholders; remove on focus.

Cards

White surfaces on light gray background. Shadow 0 10px 28px rgba(17,24,39,0.12); radius 8px.

  • Internal padding 16px; consistent 8px gaps between text blocks.
  • Hover states: slight lift and stronger shadow for clickable cards.

Navigation

Sticky 64px header on white; active or hover teal underline. "Get Started" may use Energy Orange.

  • Logo left at about 40px high; links spaced 24px.
  • Mobile: hamburger reveals vertical menu on white surface.

Layouts and CTA

Homepage, technology, docs, and CTA patterns.

Use the same system across pages: generous white space, alternating bands, and consistent typography.

Homepage

  • Hero: H1 in Kalista Blue, subhead dark text, orange or blue CTA.
  • Features: 3-up cards on white; use teal icons.
  • Band break: dark blue section with white text for emphasis.

Technology

  • Two-column alternating text or diagram; captions in small italics.
  • Diagrams use blue and teal lines; code blocks with light gray background.
Prioritize readability; keep bright accents for key callouts only.

Developer docs

  • Sidebar nav with active blue indicator; content width optimized for 70 characters.
  • Callouts with colored left borders (teal for info, orange for tips, red for errors).
  • Pagination links with arrow icons in blue on hover.
